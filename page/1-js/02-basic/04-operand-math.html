연산자와 연산

수학 연산자 : +, -, *, /, %(나머지), **(거듭제곱)

'1' + 2 == "12", 2 + '1' == "21", 2 + 2 + "1" == "41"
6 - '2' == 4, '6'/'2' == 3,

앞에 +를 붙이면 숫자로 바꾸어줌

+true == 1, +"" == 0

비트 연산자

  • 비트 AND ( & )
  • 비트 OR ( | )
  • 비트 XOR ( ^ )
  • 비트 NOT ( ~ )
  • 왼쪽 시프트(LEFT SHIFT) ( << )
  • 오른쪽 시프트(RIGHT SHIFT) ( >> )
  • 부호 없는 오른쪽 시프트(ZERO-FILL RIGHT SHIFT) ( >>> )

참고문서 : 비트 연산자

비교 연산자

== 는 값만 비교하고, === 은 타입까지 비교

null == undefined ; true, null === undefined; false

null >= 0이 참을 반환하는 이유는 (기타 비교 연산자의 동작 원리에 따라) null이 숫자형으로 변환돼 0이 되기 때문

함정 피하기

  • 일치 연산자 ===를 제외한 비교 연산자의 피연산자에 undefined나 null이 오지 않도록
  • undefined나 null이 될 가능성이 있는 변수가 <, >, <=, >= 의 피연산자가 되지 않도록 주의
  • undefined나 null이 될 가능성이 있으면 별도의 처리 코드를 추가해야함

불린형으로 변환

if(...) 문은 괄호 안의 표현식을 평가하고 그 결과를 불린으로 반환

0,"", null, undefined, NaN은 false

3항 연산 ; condition ? "true": "false"

nullish 병합 연산자 ??

 a ?? b ; a가 null이나 undefined면 b
let firstName = null;
let lastName = null;
let nickName = "바이올렛";

// null이나 undefined가 아닌 첫 번째 피연산자
alert(firstName ?? lastName ?? nickName ?? "익명의 사용자"); // 바이올렛

'??'와 '||'의 차이

let height = 0;
alert(height || 100); // 100
alert(height ?? 100); // 0

||는 height가 0 이므로 false여서 100이 됨

이래서 초기화 시에 주로 || 대신 ??를 많이 사용함

??와 ||, && 동시 사용안됨(사용하고 싶으면 ||, &&를 괄호로 묶어서 사용)
